STABLE MANIFEST UPDATE — LOADER 1.0.3

Исправлено:
- версия лоадера и обоих manifest — 1.0.3;
- основной адрес возвращён на рабочий raw.githubusercontent.com из первой версии;
- к каждому запросу добавляется уникальный параметр против локального кэша;
- GitHub API полностью убран из запуска, потому что на части Windows он вызывал 12002/12030;
- jsDelivr используется только как резервный источник;
- перед каждым запуском manifest повторно проверяется онлайн;
- локальный кэш не разрешает запуск без онлайн-проверки;
- maintenance и несовпадающая версия по-прежнему блокируют запуск.

Установка:
1. Распаковать архив поверх полного проекта с заменой.
2. Закрыть Visual Studio и старые NeverloseLoader.exe.
3. Запустить build-release.bat.
4. Запускать build/Release/NeverloseLoader.exe.
5. Создать Release loader-1.0.3 с новым EXE.
6. После этого отправить manifest в main.

Изменение JSON применяется только после успешного git push origin main.
