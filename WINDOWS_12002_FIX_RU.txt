WINDOWS ERROR 12002 HOTFIX — LOADER 1.0.3

Причина была в сетевой логике лоадера:
- GitHub API использовался как единственный быстрый путь;
- таймауты были слишком агрессивными;
- автоматическое определение прокси выполнялось раньше прямого подключения;
- на некоторых Windows WinHTTP не успевал получить ответ и возвращал 12002.

Исправлено:
- сначала выполняется прямое подключение без прокси;
- затем используется только настроенный системный прокси без медленного WPAD auto-discovery;
- таймауты разделены для маленького manifest и больших файлов;
- добавлен Connection: close для коротких запросов;
- GitHub API остаётся первым и самым свежим источником;
- raw.githubusercontent.com используется вторым;
- jsDelivr используется третьим резервным источником;
- при успешном GitHub API изменения manifest применяются практически сразу;
- версия лоадера остаётся 1.0.3.

Установка:
1. Распаковать архив поверх полного проекта с заменой.
2. Закрыть Visual Studio и все старые NeverloseLoader.exe.
3. Запустить build-release.bat.
4. Запускать только build/Release/NeverloseLoader.exe.
5. Создать Release loader-1.0.3, затем выполнить git push manifest.

После push перезапустить лоадер или нажать Retry online verification.
